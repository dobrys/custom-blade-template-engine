# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

A custom PHP micro-framework that uses Laravel's standalone `illuminate/view`,
`illuminate/events` and `illuminate/translation` packages to provide Blade
templating outside of a full Laravel app. There is no Eloquent/database
abstraction beyond a small `PDO` factory. PSR-4 autoload root: `App\` → `src/`.

## Commands

```bash
composer install      # PHP dependencies (vendor/)
npm install            # JS dependencies (node_modules/)
npm run dev             # Vite dev server
npm run build           # Vite build -> themes/default/assets/js/all.js (entry: svelte/svelte-all.js)
npm run preview
```

There is no real test suite (`npm test` is a placeholder that exits with an
error) and no lint/typecheck command configured. The `.idea/engine.iml`
references a `spec/` test source folder but it does not exist yet.

## Request lifecycle

1. **`public/index.php`** (or root `index.php`, both work — `.htaccess`
   rewrites everything to `index.php`) requires `router.php`.
2. **`router.php`** requires `init.php`, defines the static route table, and
   dispatches `App\Nav\Router`.
3. **`init.php`** bootstraps everything global:
   - Loads `.env` via `Dotenv`, defines the `SK` constant from `JWT_SECRET`.
   - Starts the session (`SessionManager`), checks `SessionManager::isLoggedIn()`.
   - Loads `config.php` (array config, `$config`) and `languages.php` (`$languages`).
   - Runs `LanguageDetector`, persists language/locale/direction to session.
   - Builds `App\App` → instantiates `BladeEngine` as `$blade` (also pushed to `$GLOBALS['blade']`).
   - Builds the initial `NavBuilder` and assigns globals to Blade: `nav`,
     `currentUrl`, `languages`, `site_language`, `text_direction`, `siteURL`,
     `is_logged_in`, `current_path`.
4. **`App\Nav\Router::dispatch()`**:
   - `resolveRoute()` trims the request URI; `''` → `home`; anything starting
     with `themes` → `assets`; otherwise validates the path against
     `^[a-zA-Z0-9\/_\-]+(\.php)?$` (400 Bad Request if it fails) and strips a
     trailing `.php`.
   - `matchSpecialRoutes()` — regex routes loaded from `routes/map.php` (does
     not currently exist; generated by `utils/generate_routes_map.php` when
     needed). Each entry is `[file, paramNames, middlewares]`; regex captures
     are mapped into `$_GET`/`$_REQUEST`.
   - `matchStaticRoute()` — looks up the trimmed route in the `$routes` array
     defined in `router.php` (keys like `home`, `login`, `profile`, ...).
   - Both paths run any configured middleware, then `refreshNavAndAuth()`
     (rebuilds nav + `is_logged_in` against the *current* route), then
     `require_once` the route's PHP file and `exit`.
   - No match → 404 via `errors.404` view.
   - Every non-special URI is recorded into the session as `next_page`
     (`config['next_uri_var']`) for post-login redirects, except the URIs
     listed in `config['special_uri']`.

Route files in `routes/*.php` are plain scripts: they `global $blade;`,
optionally call `__()` for translated strings, then `$blade->assign(...)` /
`$blade->display('pages.xxx')`.

## Theming & views

- `config.php` → `theme` (`default` or `astro`) selects the active theme.
- `BladeEngine` (`src/BladeEngine.php`) registers **two** view paths with
  Laravel's `FileViewFinder`: `themes/{theme}/` and `views/`. A view name
  like `pages.home` resolves to `themes/{theme}/pages/home.blade.php` first,
  falling back to `views/pages/home.blade.php`. Layouts/partials/errors
  follow the same dual-location pattern (`layout.default`, `partials.header`,
  `errors.404`).
- Custom Blade directives: `@dump`, `@dd` (Symfony VarDumper), `@asset(...)`
  and `@themeAsset(...)` (wrap the `asset()` / `theme_asset()` helpers).
- `BladeEngine::renderString()` compiles/evals an ad-hoc Blade string
  (auto-prefixes bare `{{ var }}` placeholders with `$`) — used for
  rendering dynamic snippets (e.g. CMS content) outside the view files.
- `themes/{theme}/assets/...` is served by `routes/assets.php`, which
  resolves the path under the theme dir with `realpath()` and rejects
  anything escaping the theme directory (path traversal guard) before
  streaming it with a manually maintained MIME map.

## Internationalization

- `languages.php` maps subdomain prefixes (`bg`, `de`, `ro`, ...) to
  Laravel-style locales (`bg_BG`, `de_DE`, ...) plus an optional `dir`
  (`rtl`) for Arabic locales. `default` key picks the fallback when no
  subdomain matches.
- `LanguageDetector` derives language/locale/direction from
  `HTTP_HOST` (stripping a leading `www`), then lets a saved
  `$_SESSION['app_locale']` override it. `persist()` writes
  `app_locale`/`app_language`/`app_direction` back to the session.
- `routes/set-lang.php` is the language-switch endpoint
  (`?locale=xx_XX&back=/path`); it validates the locale via
  `LanguageDetector::isValidLocale()` and only allows relative `back` paths.
- `config['translator']` selects `GettextTranslator` (reads `lang/{locale}.mo`
  or `.po`, falls back to `en` if the locale file is missing) or
  `LaravelTranslator` (Laravel `FileLoader`, throws if the lang dir is
  missing). The global `__()` helper (`src/helpers/translator.php`) proxies
  to whichever translator was injected into `$GLOBALS['translator']`.

## Authentication

- Hybrid session + JWT: `App\Auth\AuthJwt` (firebase/php-jwt, HS256, secret =
  `JWT_SECRET` env / `SK` constant) stores an `auth_token` cookie with
  `member_msisdn`, `service`, `user_id`, `provider`, `domain`,
  `issued_at`/`expires_at` (default lifetime 608400s ≈ 1 week + 1h).
- `App\Middleware\AuthMiddleware` → `App\Auth\AuthService::handle()`:
  - already authenticated (session + valid JWT) → pass through.
  - valid JWT but no session → restore session from JWT.
  - `$_REQUEST['public_uuid']` or `$_REQUEST['msisdn']` present → look the
    member up via the configured `AuthProviderInterface` providers and log in
    if active.
  - expired/missing JWT → redirect to `/login` (with a
    `redirect_reason` flashed to session).
- Provider selection in `AuthMiddleware` is environment-based:
  `config['env'] === 'development'` → `DummyProvider` (in-memory fixture
  members); otherwise `NthProvider` → `App\Models\NthMember` (raw PDO via
  `DatabaseFactory`, reads `src/assets/config/face.php` for DB credentials —
  not present in this repo, must be provisioned per environment).
- `routes/signin.php` is the post-login bounce: redirects to whatever URI was
  captured in `$_SESSION[config['next_uri_var']]` (set by the router on every
  non-special request).
- `routes/logout.php` clears the session and, if
  `JWT_REMOVE_ON_LOGOUT=true`, also clears the JWT cookie.

## Helper functions (`src/helpers/functions.php`, autoloaded via `src/App.php`)

- `asset($path)` — absolute HTTPS URL under `/assets/`.
- `theme_asset($path)` — absolute HTTPS URL under `/themes/{current_theme}/assets/`
  (current theme comes from `$GLOBALS['current_theme']`, set by `BladeEngine`).
- `input($key, $default, $source = 'request', $sanitize = true)` — reads
  from `$_POST`/`$_GET`/`$_REQUEST`, HTML-escapes by default (including
  recursively for arrays).
- `env($key, $default)` — `$_ENV` → `$_SERVER` → `getenv()` → default.
- `__($key, $replace, $locale)` — translation helper, defined in both
  `src/helpers/translator.php` and `src/translator.php` (identical; both are
  required, guarded by `function_exists`).

## Frontend build

- Vite + Svelte (custom elements mode). `vite.config.js` builds
  `svelte/svelte-all.js` into `themes/default/assets/js/all.js` (single
  bundle, `emptyOutDir: false`). The `svelte/` source directory does not
  exist yet in this checkout — create it before running `npm run build`/`dev`
  if working on Svelte components.

## Notes / gotchas

- `__globals.php` at the repo root is currently dead code (not `require`d
  anywhere) and defines a *different*, hardcoded `SK` constant — don't
  confuse it with the real `SK` defined in `init.php` from `JWT_SECRET`.
- Adding a new top-level page requires **both** a route entry in the
  `$routes` array in `router.php` and a Blade view under `views/pages/` or
  `themes/{theme}/pages/`. Several nav entries (`/profile`, `/single`,
  `/terms`, `/settings`, `/billing`) and the `subscribe` route
  (`landing/mobixon/index.php`) are referenced but have no backing
  route/file yet.
- Comments and many identifiers throughout the codebase are in Bulgarian.
